# JIRA操作の使用例

ClaudeとMCPサーバー経由でJIRAを操作する実例を紹介します。

## 基本的な使い方

### 課題の検索

**ユーザー入力:**
```
プロジェクトOPEの未完了チケットを検索して
```

**Claudeの動作:**
- `jira_search_issues` ツールを使用
- JQL: `project = OPE AND status != Done`
- 検索結果を整形して表示

**期待される結果:**
```
OPEプロジェクトの未完了チケットを検索しました。以下の課題が見つかりました:

1. OPE-123: ログイン画面のバグ修正
   - ステータス: In Progress
   - 担当者: 田中太郎
   - 優先度: High

2. OPE-124: API仕様書の更新
   - ステータス: To Do
   - 担当者: 佐藤花子
   - 優先度: Medium
...
```

### 課題の詳細取得

**ユーザー入力:**
```
OPE-123の詳細を教えて
```

**Claudeの動作:**
- `jira_get_issue` ツールを使用
- 課題キー: `OPE-123`
- 詳細情報を読みやすく整形

**期待される結果:**
```
OPE-123の詳細情報:

タイトル: ログイン画面のバグ修正
タイプ: Bug
ステータス: In Progress
優先度: High
担当者: 田中太郎
作成日: 2026-02-15
更新日: 2026-03-01

説明:
ユーザー名に特殊文字を入力すると500エラーが発生する問題を修正する。

コメント:
- 2026-02-16 佐藤花子: 再現確認しました
- 2026-02-20 田中太郎: 修正対応中です
```

## 応用的な使い方

### 複雑なJQLクエリ

**ユーザー入力:**
```
先週作成された優先度Highのバグで、まだ誰にも割り当てられていないチケットを検索して
```

**Claudeの動作:**
- JQLを生成: `project = OPE AND type = Bug AND priority = High AND assignee is EMPTY AND created >= -7d`
- `jira_search_issues` ツールを使用

### 課題の作成

**ユーザー入力:**
```
バグチケットを作成して。
プロジェクト: OPE
タイトル: ログアウト後にセッションが残る
優先度: Medium
説明: ログアウトボタンを押した後もセッションCookieが残っており、ブラウザの戻るボタンで認証状態に戻れてしまう。
```

**Claudeの動作:**
- `jira_create_issue` ツールを使用
- パラメータを整形して送信

**期待される結果:**
```
チケットを作成しました: OPE-125

詳細:
- タイトル: ログアウト後にセッションが残る
- タイプ: Bug
- 優先度: Medium
- ステータス: To Do
- URL: https://your-domain.atlassian.net/browse/OPE-125
```

### コメントの追加

**ユーザー入力:**
```
OPE-123に「再現テスト完了。問題なし」とコメントを追加して
```

**Claudeの動作:**
- `jira_add_comment` ツールを使用
- コメントを追加

**期待される結果:**
```
OPE-123にコメントを追加しました。
```

### ステータスの変更

**ユーザー入力:**
```
OPE-123のステータスを「完了」に変更して
```

**Claudeの動作:**
1. `jira_get_transitions` ツールで利用可能なトランジションを取得
2. 「完了」に対応するトランジションIDを特定
3. `jira_transition_issue` ツールでステータスを変更

**期待される結果:**
```
OPE-123のステータスを「完了」に変更しました。
```

## 実用的なワークフロー例

### スプリント準備

**ユーザー入力:**
```
次のスプリント用のタスクを準備したい。
1. 優先度HighとMediumの未完了バグを全部リストアップ
2. それぞれの担当者と最終更新日も教えて
3. 1週間以上更新されていないものは別途教えて
```

**Claudeの動作:**
1. JQLで該当するバグを検索
2. 結果を整形して表示
3. 更新日でフィルタリング

### レポート作成支援

**ユーザー入力:**
```
今週完了したタスクの一覧を作って、Confluenceに「週次レポート」というページを作成して
```

**Claudeの動作:**
1. JQLで今週完了したタスクを検索
2. 結果をConfluence形式に整形
3. `confluence_create_page` ツールでページ作成

## よくあるJQLパターン

### 自分が担当している未完了タスク
```
assignee = currentUser() AND status != Done
```

### 特定期間に作成された課題
```
created >= "2026-03-01" AND created <= "2026-03-31"
```

### 特定ラベルの課題
```
labels = "urgent" OR labels = "security"
```

### 複数プロジェクトの検索
```
project in (OPE, DEV, QA) AND status = "In Progress"
```

## Tips

1. **自然な日本語で指示**: 「〜を検索して」「〜を作成して」のように自然な言葉で指示できます
2. **段階的な操作**: 複雑な操作は段階的に指示することで、各ステップを確認できます
3. **情報の要約**: Claudeは長い検索結果を自動的に要約して表示します
4. **エラーハンドリング**: 存在しないチケットや権限エラーは自動的に検出されます

## 注意事項

- JQLの構文が不明な場合は、Claudeに「〜のようなチケットを検索したい」と自然言語で伝えてください
- 課題の削除は現在サポートしていません（APIの制限）
- 大量のチケットを作成する場合は、バッチ処理を検討してください
