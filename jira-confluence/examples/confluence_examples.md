# Confluence操作の使用例

ClaudeとMCPサーバー経由でConfluenceを操作する実例を紹介します。

## 基本的な使い方

### スペース一覧の取得

**ユーザー入力:**
```
Confluenceのスペース一覧を取得して
```

**Claudeの動作:**
- `confluence_list_spaces` ツールを使用
- アクセス可能なスペースをリスト表示

**期待される結果:**
```
アクセス可能なConfluenceスペース:

1. DEV (開発ドキュメント)
2. OPE (運用手順書)
3. QA (品質保証)
4. HR (人事)
...
```

### ページの検索

**ユーザー入力:**
```
開発スペースで「API仕様」に関するページを検索して
```

**Claudeの動作:**
- `confluence_search_pages` ツールを使用
- パラメータ: `spaceKey: "DEV"`, `title: "API仕様"`

**期待される結果:**
```
DEVスペースで「API仕様」に関するページが見つかりました:

1. REST API仕様書 v2.0
   - ページID: 123456
   - 最終更新: 2026-02-28

2. GraphQL API仕様
   - ページID: 123457
   - 最終更新: 2026-01-15
```

### ページコンテンツの取得

**ユーザー入力:**
```
ページID 123456の内容を取得して要約して
```

**Claudeの動作:**
- `confluence_get_page` ツールを使用
- コンテンツを取得して要約

**期待される結果:**
```
「REST API仕様書 v2.0」の要約:

このページは認証、エンドポイント、レスポンス形式について記載されています。

主な内容:
- 認証方式: JWT Bearer Token
- ベースURL: https://api.example.com/v2
- 主要エンドポイント: /users, /projects, /tasks
- レート制限: 1000リクエスト/時間
```

## 応用的な使い方

### ページの作成

**ユーザー入力:**
```
DEVスペースに「新機能リリースノート v2.1」というタイトルでページを作成して。

内容:
# リリース日
2026-03-10

# 新機能
- ダークモード対応
- パフォーマンス改善（読み込み速度50%向上）
- 新しいダッシュボード機能

# バグ修正
- ログイン画面のエラー修正
- セッション管理の改善
```

**Claudeの動作:**
1. DEVスペースの情報を取得（spaceIdを特定）
2. Markdown形式からConfluence Storage Formatに変換
3. `confluence_create_page` ツールでページ作成

**期待される結果:**
```
ページを作成しました: 新機能リリースノート v2.1

詳細:
- スペース: DEV
- ページID: 789012
- URL: https://your-domain.atlassian.net/wiki/spaces/DEV/pages/789012
```

### ページの更新

**ユーザー入力:**
```
ページID 123456に「更新履歴」セクションを追加して、今日の日付で「エラーハンドリングのセクションを追加」と記載して
```

**Claudeの動作:**
1. `confluence_get_page` で現在のコンテンツとバージョン番号を取得
2. 既存コンテンツに新しいセクションを追加
3. `confluence_update_page` ツールで更新

**期待される結果:**
```
ページを更新しました: REST API仕様書 v2.0
新しいバージョン: 5
```

### ページ階層の確認

**ユーザー入力:**
```
「開発ガイド」ページの子ページ一覧を表示して
```

**Claudeの動作:**
- まず「開発ガイド」ページを検索してIDを取得
- `confluence_get_child_pages` を使用（※未実装の場合は別の方法で取得）

## 実用的なワークフロー例

### ドキュメントの一括作成

**ユーザー入力:**
```
新しいプロジェクト「PROJECT-X」のドキュメントセットを作成したい。
DEVスペースに以下のページを作成して:
1. PROJECT-X 概要
2. 技術仕様書
3. 運用手順書
```

**Claudeの動作:**
1. 親ページ「PROJECT-X 概要」を作成
2. その配下に子ページとして「技術仕様書」「運用手順書」を作成
3. 各ページにテンプレート的な見出しを設定

### JIRAからConfluenceへのレポート作成

**ユーザー入力:**
```
今週完了したJIRAチケットをまとめて、Confluenceに週次レポートページを作成して
```

**Claudeの動作:**
1. `jira_search_issues` で今週完了したチケットを検索
2. 結果を整形してConfluence Storage Format に変換
3. `confluence_create_page` で週次レポートページを作成

**期待される結果:**
```
週次レポートページを作成しました。

完了したチケット: 15件
- バグ修正: 8件
- 新機能: 5件
- タスク: 2件

URL: https://your-domain.atlassian.net/wiki/spaces/DEV/pages/XXXXX
```

### ドキュメントの検索と更新

**ユーザー入力:**
```
「API仕様」で検索して、見つかったページ全部に「最終レビュー日: 2026-03-04」というラベルを追加して
```

**Claudeの動作:**
1. `confluence_search_pages` で検索
2. 各ページを順番に更新

## Confluence Storage Format について

Confluenceのページコンテンツは「Storage Format」というHTML風の形式で保存されます。

### 基本的な例

**見出し:**
```html
<h1>大見出し</h1>
<h2>中見出し</h2>
```

**段落:**
```html
<p>これは段落です。</p>
```

**リスト:**
```html
<ul>
  <li>項目1</li>
  <li>項目2</li>
</ul>
```

**リンク:**
```html
<a href="https://example.com">リンク</a>
```

**テーブル:**
```html
<table>
  <tr>
    <th>ヘッダー1</th>
    <th>ヘッダー2</th>
  </tr>
  <tr>
    <td>データ1</td>
    <td>データ2</td>
  </tr>
</table>
```

### Claudeによる自動変換

Markdownで指示しても、Claudeが自動的にStorage Formatに変換してくれます。

**ユーザー入力（Markdown）:**
```
# タイトル
## サブタイトル
- リスト項目1
- リスト項目2
```

**Claudeが変換:**
```html
<h1>タイトル</h1>
<h2>サブタイトル</h2>
<ul>
  <li>リスト項目1</li>
  <li>リスト項目2</li>
</ul>
```

## Tips

1. **自然な日本語で指示**: 「〜のページを作成して」「〜を検索して」のように指示できます
2. **Markdownで記述**: ページ内容はMarkdownで書いても自動変換されます
3. **階層構造**: `parentId` を指定することでページの親子関係を作成できます
4. **バージョン管理**: ページ更新時は自動的にバージョンが管理されます

## 注意事項

- ページの削除は現在サポートしていません
- 大量のページを一度に作成する場合は、APIレート制限に注意
- 添付ファイルのアップロードは別途実装が必要
- マクロ（展開可能なセクションなど）の挿入は Storage Format の知識が必要
